## 11. Glossary of claims used in this report

- **REAL occurrence** — a regex match found in code after the heuristic
  stripping of `--` line comments, `/- ... -/` (and `/-! ... -/`) block
  comments, and `"..."` string literals. Comments and string literals are
  replaced with spaces (preserving line numbers) before searching, so a
  `sorry` mentioned in prose or inside a string is excluded from REAL
  counts.

- **RAW occurrence** — a regex match found anywhere in the source text,
  before stripping. Counts include matches inside comments and string
  literals. Reported alongside REAL to make the heuristic's effect visible.

- **Primary scope / Secondary scope / Other** — top-level directories of the
  target repo, classified per `horizon_repo.yaml`. Primary is the focus of
  the audit; secondary is reported but kept distinct (release artefacts,
  ancillary trees); other captures everything else (root files, etc.).

- **Declaration vs Usage** (target identifiers) — heuristic line
  classification. A line is `declaration` when, after optional attributes
  and modifiers, it begins with a Lean keyword (`theorem`, `lemma`, `def`,
  `instance`, `axiom`, `abbrev`, `structure`, `class`, `inductive`)
  immediately followed by the searched identifier; otherwise `usage`.

- **Watchlist (unresolved)** — names listed in `proof_targets.yaml` that
  were not found in the primary scope. Still scanned in case they appear
  later. Their absence is descriptive, not normative: the audit makes no
  claim about whether they should exist.

- **Namespaces of interest** — directory prefixes (relative to the target
  repo) whose declarations are enumerated as a group. Treated as a
  filesystem grouping, not a Lean `namespace` block parse.

- **Heuristic, not a Lean parser** — pattern detection and identifier
  classification are regex-based with a small state machine for comments
  and strings. Edge cases (char literals `'x'`, raw strings, interpolated
  strings `s!"..."`, syntax extensions) may produce false-positive or
  false-negative classifications. The tool flags this rather than hiding
  it.
